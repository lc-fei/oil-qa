# 会话记忆规格

## Purpose
定义 C 端问答的会话级对话记忆能力，包括滚动摘要、待摘要溢出轮次、最近原文轮次、结构化记忆 key、问题理解与 Prompt 注入，以及编排归档中的记忆快照。

## Requirements
### Requirement: 会话级对话记忆

系统 MUST 只在当前 QA 会话内维护对话记忆，且不得跨会话共享。

#### Scenario: 同一会话使用记忆

- **WHEN** 用户在同一会话中继续提问
- **THEN** 系统必须使用该会话维护的记忆。

#### Scenario: 不同会话隔离

- **WHEN** 同一用户创建或打开另一个会话
- **THEN** 系统不得使用之前会话中的记忆。

### Requirement: 滚动摘要记忆

系统 MUST 将会话记忆表达为滚动摘要、待摘要溢出轮次、最近原文轮次和结构化记忆 key。

#### Scenario: 构建问答记忆

- **WHEN** C 端问答请求启用上下文模式
- **THEN** 系统必须基于当前会话摘要、待摘要溢出轮次、最新配置数量的原文问答轮次和记忆 key 构建记忆上下文。

#### Scenario: 上下文模式关闭

- **WHEN** C 端问答请求关闭上下文模式
- **THEN** 系统必须跳过记忆读取和 Prompt 记忆注入。

### Requirement: 记忆参与问题理解

系统 MUST 将对话记忆上下文传递给问题理解阶段。

#### Scenario: 追问问题

- **WHEN** 用户提出类似“它的原因是什么”的追问
- **THEN** 问题理解阶段必须接收会话记忆，以便结合上下文重写或解释问题。

#### Scenario: 空记忆

- **WHEN** 当前不存在会话记忆
- **THEN** 问题理解阶段必须正常处理原始问题。

### Requirement: 记忆参与 Prompt 构建

系统 MUST 以清晰分区和有界长度将会话记忆注入答案生成 Prompt。

#### Scenario: 记忆注入 Prompt

- **WHEN** 会话记忆可用
- **THEN** Prompt 必须包含摘要、待摘要溢出轮次、记忆 key 和最近原文轮次等分区。

#### Scenario: 记忆长度受控

- **WHEN** 会话记忆超过配置的文本长度限制
- **THEN** 系统必须以确定性方式截断注入记忆，并将记忆上下文标记为已截断。

### Requirement: 结构化记忆 key

系统 MUST 维护当前会话内的结构化记忆 key。

#### Scenario: 记录关键实体

- **WHEN** 问题理解或图谱检索识别出相关实体
- **THEN** 系统必须能够将关键实体保存到会话记忆中。

#### Scenario: 记录用户偏好

- **WHEN** 用户在当前会话中明确表达偏好
- **THEN** 系统必须能够将该偏好保存为会话级记忆。

#### Scenario: 记录约束条件

- **WHEN** 用户在当前会话中表达工况、场景或回答风格等约束
- **THEN** 系统必须能够将这些约束保存为会话级记忆 key。

### Requirement: 记忆摘要维护

系统 MUST 在成功问答轮次之后维护滚动摘要，同时保留最新配置数量的原文轮次。

#### Scenario: 存在溢出轮次

- **WHEN** 成功问答产生了早于最新原文窗口且尚未摘要的轮次
- **THEN** 系统必须基于上一版摘要和这些溢出轮次生成新摘要。

#### Scenario: 摘要成功

- **WHEN** 摘要生成成功
- **THEN** 系统必须更新摘要、记忆 key、摘要游标、摘要版本，并继续保留最新配置数量的原文轮次作为原文记忆。

#### Scenario: 摘要失败

- **WHEN** 摘要生成失败
- **THEN** 系统必须保留上一版记忆状态，不得推进摘要游标，必须继续保留最近原文轮次，并且不得导致已完成的问答响应失败。

#### Scenario: 摘要不阻塞回答

- **WHEN** 成功问答后的摘要生成仍在等待或执行缓慢
- **THEN** 系统不得仅因为摘要生成而延迟非流式聊天响应或 SSE done 事件。

#### Scenario: 待摘要溢出继续携带

- **WHEN** 摘要生成尚未纳入溢出轮次
- **THEN** 系统必须在后续记忆上下文中继续携带这些待摘要溢出轮次，但受配置长度限制约束。

### Requirement: 记忆使用被归档

系统 MUST 在问答编排归档中记录每次 QA 请求实际使用的记忆快照。

#### Scenario: 记忆快照归档

- **WHEN** 问答请求使用了会话记忆
- **THEN** 系统必须记录摘要快照、记忆 key 快照、使用的消息标识、摘要游标、最近窗口大小、待摘要溢出数量和截断标记。

#### Scenario: 未使用记忆

- **WHEN** 没有使用会话记忆
- **THEN** workflow 和归档必须保持有效，并能表示空记忆上下文。

### Requirement: 现有 C 端 API 兼容

系统 MUST 保持现有 C 端问答请求参数兼容，同时在内部改进记忆行为。

#### Scenario: 现有客户端请求

- **WHEN** 现有客户端调用 `POST /api/client/qa/chat` 或 `POST /api/client/qa/chat/stream` 且不传入新的记忆参数
- **THEN** 系统必须使用默认记忆行为成功处理该请求。
